# 架構決策

## 導航結構
底部 Tab Bar 共四個 tab：
1. 首頁（動態牆）
2. 探索（搜尋、分類瀏覽）
3. 通知
4. 個人頁面

發文按鈕：固定顯示在右下角的浮動按鈕，任何頁面都可以點擊發文。

## 帳號等級系統
參考 Yahoo 知識+，依參與程度累積點數，分為五個等級：
- Lv1 新人：0 點，剛加入，功能受限
- Lv2 社員：100 點，基本參與
- Lv3 活躍：500 點，積極貢獻
- Lv4 資深：2000 點，長期參與
- Lv5 傳奇：5000 點，頂級貢獻者

點數來源：
- 發文：+10 點
- 回覆：+5 點
- 發文被按讚：+3 點/個
- 回覆被按讚：+2 點/個
- 回覆被標記為最佳回覆：+20 點

等級限制：
- Lv1：每日最多發文 3 篇、無法附圖、無法使用匿名功能
- Lv2：每日最多發文 5 篇、可附 1 張圖、可匿名
- Lv3 以上：無發文數量限制、可附 3 張圖、所有功能解鎖

## 匿名機制
- 發文和回覆都可選擇匿名（需 Lv2 以上）
- 匿名時顯示為「匿名用戶」，不顯示帳號等級和身份標籤
- 管理員可查看匿名發文者的真實身份（用於違規處理）

## 身份標籤
每個用戶名字旁顯示身份標籤：
- 現役社員：顯示「社員」藍色標籤
- 校友：顯示畢業屆別，例如「2022屆」紅色標籤
- 一般用戶：不顯示標籤
- 管理員：顯示「管理員」金色標籤
- 匿名發文時不顯示任何標籤

## 按讚機制
- 每篇發文和每則回覆都可以按讚或倒讚
- 每個用戶對同一則內容只能按一次
- 按讚數和倒讚數都公開顯示
- 按讚數影響帳號等級點數計算

## Instagram 獎勵機制
- 每個月統計所有校友用戶的回覆被按讚總數
- Story 獎勵：當月被按讚數排名前 5 的校友可獲得 ROCSAUT Instagram Story 曝光
- 貼文獎勵：當月被按讚數排名第 1 的校友可獲得 ROCSAUT Instagram 貼文曝光
- 由管理員在 Admin 介面手動執行每月評選並通知得獎者
- 獎勵只針對校友身份的用戶，現役社員不參與

## API 架構原則
- 所有業務邏輯放在後端 API Routes
- 前端只負責呈現和互動
- 認證使用 JWT Bearer Token
- 初期複用小社團平台後端，在現有 Next.js 專案中新增 /api/community/ 前綴的路由

## 資料庫設計原則
- 大社團平台的資料表使用 community_ 前綴避免與小社團平台衝突
- 初期共用 Supabase 專案，長期目標是獨立
